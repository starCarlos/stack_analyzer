## 1. 项目定位

本项目是 **Web 壳 + 数据引擎 + 决策大脑** 三合一系统，统一承载：
- 数据刷新、质量检查、归档治理

### 2.1 为什么保留 DDD 分层（domain / infrastructure / services）

当前仓库是“**主路径 app/** + **DDD 路径（顶层 domain/infrastructure/services 等）**”并存形态。  
选择 DDD 分层的目的，是把“业务规则”与“技术细节”解耦，便于后续替换存储、扩展接口、做回测/实盘并行。

- `domain/`：放领域对象与核心业务语义（不依赖 Web/DB 框架）
- `services/`：放用例编排（例如推荐、回放、看板聚合），负责“做什么流程”
- `infrastructure/`：放外部依赖实现（SQLite、第三方数据源等），负责“怎么连外部系统”
- `api/`、`pipeline/`、`core/`（顶层）：分别承载接口暴露、流程调度、配置与横切能力

这样分层后，业务演进时可做到：
1) 领域规则稳定；2) 基础设施可替换；3) 接口层可多形态复用（Web/API/CLI）。

## 3. 每日流水线（16 步，含 critical/warn）

入口：`app/scheduler/jobs.py::daily_pipeline`

1. 刷新数据（含板块池同步）**[critical]**
2. 数据质量检查 **[warn]**
3. 抓取新闻 **[warn]**
4. 信号包 + LLM 打分 **[critical]**
5. 信号质量检查 **[warn]**
6. 生成预测（大盘/个股/全板块）**[critical]**
7. 评估到期预测 **[critical]**
8. 误差诊断 **[warn]**
9. 自动学习调参 **[warn]**
10. 稳健性验证 **[warn]**
11. 候选筛选 + 定性风控 **[warn]**
12. Advisor 多角色分析 + 一致性回写 signal **[warn]**
13. 纸面交易 **[warn]**
14. 学习报告 **[critical]**
15. 推送 **[warn]**
16. 归档治理 **[warn]**

失败策略：
- `critical` 失败：流水线终止；首页继续展示最近一次成功运行结果，并显示“数据截至 YYYY-MM-DD”的告警条（待前端告警条细化）。
- `warn` 失败：记录错误并继续后续步骤。

并落库：
- `pipeline_runs`
- `quality_check_logs`
- `signal_packs`
- `archive_logs`

## 4. 首页功能（已升级）

首页 `/` 当前包含：

1. **市场温度计顶栏**（6 指标）
   - 沪深300、创业板指、北向净流入、两市成交额、10Y 国债、USD/CNY
   - 含涨跌语义色标

2. **预测区**
   - 大盘预测
   - 个股 TOP
   - 全板块预测（显示覆盖数量与模式）

3. **可执行建议**
   - 个股：入场价/止盈价/止损价/仓位
   - 板块：入场价/止盈价/止损价/仓位
   - 大盘：加仓/减仓/观望 + 目标总仓位 + 触发条件
   - 计算口径（当前）：  
     - `止损价 = 入场价 × (1 - STOP_LOSS_PCT)`  
     - `止盈价 = 入场价 × (1 + 目标收益系数)`  
     - `建议仓位 = min(单只上限, 置信度×基础系数)`  
     - 组合层总仓位受 `max_total_pct` 约束（超限时按置信度降序截断）

4. **跨市场事件影响表**
   - 美股/日本/欧洲事件 → A股板块和标的映射

5. **学习摘要 + 今日优先级**
   - 昨日/周/月命中率
   - 参数变更
   - P0/P1/P2 今日操作优先级
   - 分级定义：  
     - `P0`：已持仓触发止损/止盈等立即动作  
     - `P1`：高置信度新建仓或调仓  
     - `P2`：观望/跟踪，不立即执行

## 5. 全板块模式（已支持）

配置：`FULL_SECTOR_MODE=auto|akshare|config`

- `auto`：优先 AKShare → 缓存 → fallback 全板块清单
- `akshare`：强制 AKShare（失败兜底）
- `config`：仅用 `config/symbols.yaml`

接口：`GET /api/system/sectors`

## 6. LLM 接入（已接通）

LLM 客户端：`app/llm/client.py`

支持：
- `chat_completions` 与 `responses` 双模式（`LLM_API_STYLE`）

- `LLM_BASE_URL`
- `LLM_API_KEY`
- `LLM_MODEL`
- `LLM_API_STYLE`
- `LLM_TIMEOUT`
- `LLM_JSON_TIMEOUT`
- `LLM_JSON_MAX_TOKENS`
- `LLM_JSON_RETRY`
- `LLM_DAILY_BUDGET_TOKENS`（规划中，用于成本闸门）

已接入链路：
- `news_scorer`

## 7. 学习闭环与回滚关系（已从骨架升级）

### 7.1 历史学习
`learner.learn_from_history(start, end)`：
- 按历史交易日生成 `as_of_date` 预测
- 写 `forecast_evaluations`
- 调用 `learn_from_evaluations()`

### 7.2 自动学习
`learn_from_evaluations()`：
- 基于评估命中率调整 `forecast_model.yaml` 权重
- 写 `parameter_audit`
- 写 `learning_log`
- 最小样本门槛：`LEARN_MIN_SAMPLES=20`

### 7.3 历史回放
`replayer.replay()`：
- 调历史学习
- 写 `replay_sessions`

### 7.4 稳健性验证与回滚
- 已实现：步骤9调参后，步骤10验证失败会自动触发参数回滚。
- 回滚实现：
  - 回滚 `forecast_model.yaml` 对应参数到旧值
  - 回滚后权重重新归一化
  - 写入 `parameter_audit`（`source=rollback`）
  - 在学习页与学习 API 输出“回滚事件”提示

### 7.5 前视偏差控制
- 在 `learn_from_history` 中已加入断言：`max_visible_date <= as_of_date`。
- 作用：避免历史回放“看到未来数据”。

## 7.6 Advisor 与风控规则（补充）

Advisor（当前定义）：
- 角色：`fundamental/risk/policy/supply_chain`
- Persona：李迅雷/付鹏/任泽平风格
- 一致性：`role_agreement = max(同向有效票数) / 有效投票角色数`  
  （`neutral` 默认不计入有效投票分母）
- 低一致性候选降级（规划中，待回写步骤11）

风控硬规则（来自 `config/decision_rules.json`）：
- 单只上限：10%
- 总仓位上限：60%
- 日买入上限：5%
- 止损：8%
- 过滤：排除 ST、最小上市天数、市值门槛
- 大盘制动：跌破 MA20 禁新开仓（规则定义已在配置，执行增强进行中）
