## 1. 项目定位

本项目是 **Web 壳 + 数据引擎 + 决策大脑** 三合一系统，统一承载：
- 数据刷新、质量检查、归档治理
- 预测、评估、学习闭环
- 多角色 Advisor + LLM 结构化分析
- 页面展示与 API 服务

## 3. 每日流水线（16 步，含 critical/warn）

入口：`app/scheduler/jobs.py::daily_pipeline`

1. 刷新数据（含板块池同步）**[critical]**
2. 数据质量检查 **[warn]**
3. 抓取新闻 **[warn]**
4. 信号包 + LLM 打分 **[critical]**
5. 信号质量检查 **[warn]**
6. 生成预测（大盘/个股/全板块）**[critical]**
7. 评估到期预测 **[critical]**
8. 误差诊断 **[warn]**
9. 自动学习调参 **[warn]**
10. 稳健性验证 **[warn]**
11. 候选筛选 + 定性风控 **[warn]**
12. Advisor 多角色分析 + 一致性回写 signal **[warn]**
13. 纸面交易 **[warn]**
14. 学习报告 **[critical]**
15. 推送 **[warn]**
16. 归档治理 **[warn]**

失败策略：
- `critical` 失败：流水线终止；首页继续展示最近一次成功运行结果，并显示“数据截至 YYYY-MM-DD”的告警条（待前端告警条细化）。
- `warn` 失败：记录错误并继续后续步骤。

并落库：
- `pipeline_runs`
- `quality_check_logs`
- `signal_packs`
- `archive_logs`

## 5. 全板块模式（已支持）

配置：`FULL_SECTOR_MODE=auto|akshare|config`

- `auto`：优先 AKShare → 缓存 → fallback 全板块清单
- `akshare`：强制 AKShare（失败兜底）
- `config`：仅用 `config/symbols.yaml`

接口：`GET /api/system/sectors`
- 返回 mode/source/count/cache/sectors

## 6. LLM 接入（已接通）

LLM 客户端：`app/llm/client.py`

支持：
- `chat_completions` 与 `responses` 双模式（`LLM_API_STYLE`）
- 默认系统提示词（`LLM_SYSTEM_PROMPT`）
- `response_format`（json_object/json_schema）
- `responses` 网关字段兼容（`text.format.name/schema/strict`）
- JSON 失败降级重试
- 简单缓存（内存）

关键环境变量：
- `LLM_BASE_URL`
-.`LLM_API_KEY`
- `LLM_MODEL`
- `LLM_API_STYLE`
- `LLM_TIMEOUT`
- `LLM_JSON_TIMEOUT`
- `LLM_JSON_MAX_TOKENS`
- `LLM_JSON_RETRY`
- `LLM_DAILY_BUDGET_TOKENS`（规划中，用于成本闸门）

已接入链路：
- `news_scorer`
- `policy_scorer`
- `event_analyst`

备注：
- 已兼容 `responses` 网关对 `text.format.name/schema/strict` 的参数要求
- `chat_json` 在 `responses` 不支持 strict format 时会自动降级重试
- 当日预算超限时将降级为规则模式（规划中）

## 7. 学习闭环与回滚关系（已从骨架升级）

### 7.1 历史学习
`learner.learn_from_history(start, end)`：
- 按历史交易日生成 `as_of_date` 预测
- 写 `forecast_evaluations`
- 调用 `learn_from_evaluations()`

### 7.2 自动学习
`learn_from_evaluations()`：
- 基于评估命中率调整 `forecast_model.yaml` 权重
- 写 `parameter_audit`
- 写 `learning_log`
- 最小样本门槛：`LEARN_MIN_SAMPLES=20`

### 7.3 历史回放
`replayer.replay()`：
- 调历史学习
- 写 `replay_sessions`

### 7.4 稳健性验证与回滚
- 设计目标：步骤9调参后，步骤10验证失败则回滚步骤9变更。
- 当前状态：验证已接入，自动回滚机制**待补齐**（优先级高）。

### 7.5 前视偏差控制
- 在 `learn_from_history` 中已加入断言：`max_visible_date <= as_of_date`。
- 作用：避免历史回放“看到未来数据”。

## 7.6 Advisor 与风控规则（补充）

Advisor（当前定义）：
- 角色：`fundamental/risk/policy/supply_chain`
- Persona：李迅雷/付鹏/任泽平风格
- 一致性：`role_agreement = max(vote_count)/total_roles`
- 低一致性候选降级（规划中，待回写步骤11）

风控硬规则（来自 `config/decision_rules.json`）：
- 单只上限：10%
- 总仓位上限：60%
- 日买入上限：5%
- 止损：8%
- 过滤：排除 ST、最小上市天数、市值门槛
- 大盘制动：跌破 MA20 禁新开仓（规则定义已在配置，执行增强进行中）
